• 2024 — FERN / WIRED investigation: only 20% of users reported higher yields. Advisory channels farmers toward pesticides.

Appendix — Agentic architecture
A14 / 16

One cheap classifier. 11 intents. Only the right tools bound per turn.

Tool-call bloat and cross-contamination solved without fine-tuning.
1

Intent classifier (max_tokens = 8)

Lightweight OpenAI-compatible model classifies each query into one of 11 intents: weather, irrigation, pest_disease, crop_protection, diagnosis, market, fertilizer, sowing, scheme, greeting, general. Sees the last 5 turns of conversation history.

2

Per-intent system prompt + tool binding

Gemini 2.5 receives an intent-specific system prompt and only the tools relevant to that intent via bind_tools. 16 tools total; typically 2–5 bound per turn. Eliminates cross-contamination and cuts token cost vs. binding everything on every call.

3

Mandatory-tool enforcement + retry loop

cibrc_safety_check and imd_weather_check declared mandatory in config.yaml. Post-loop inspection scans the message history; if a required tool wasn't called, the agent is re-prompted up to 2 times before the answer is accepted.

4

RAG pipeline + geo resolution

LLM-driven Hinglish keyword extractor ("gehu" → wheat, "dhan" → rice) pulls ICAR ChromaDB + KCC transcript collections. Geo resolver turns (state, district) into the nearest IMD agromet station + Agmark mandi with district-neighbour fallback — making every downstream tool call location-aware.

Appendix — Chemical safety implementation
A15 / 16

Defense-in-depth. Three layers. A banned chemical cannot survive all three.

448 CIBRC chemicals loaded at startup. 88 currently banned. Earlier post-hoc classifier removed once this stack was proven sufficient.
1

Prompt guardrails

Intent-specific system prompts instruct the agent never to recommend unregistered or banned chemicals, and to cite sources. First line of defense — necessary but, on its own, not reliable.

2

Forced cibrc_safety_check tool call

Declared mandatory in config.yaml. Post-loop inspection verifies the tool was called on every chemical-adjacent query. If skipped, the agent is re-prompted up to 2 times before any answer is accepted — the tool call is not optional.

3

Post-hoc regex SafetyValidator + hard block

Precomputed banned-chemical set from the CIBRC CSV. Regex scan runs on the LLM's final answer. If a match fires in strict mode, the answer is sent back to the LLM with an explicit revision instruction; if the revision still contains banned substances, the response is replaced with a hard block.

Why three layers? A single prompt warning proved unreliable in testing. Each layer is independently observable, independently toggleable in config.yaml, and independently tested in the 426-test suite.
